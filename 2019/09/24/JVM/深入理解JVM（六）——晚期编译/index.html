深入理解JVM（六） --- 晚期编译（运行期编译）

2019-09-24 JAVA JAVA, JVM 0 评论 字数统计: 587(字) 阅读时长: 2(分)

晚期编译（运行期编译）即是编译器的后端编译，此时编译器会将早期编译（前端编译）生成的AST。

Java的编译包括前端编译、后端编译，前端编译器把*.java编译为*.class，而后端编译有即时编译器（JIT）、提前编译（AOT）等。当虚拟机发现某个方法或代码块运行的比较频繁时，就会把这部分代码标记为热点代码。并且为了在运行热点代码时提高效率，虚拟机会将这部分代码通过即时编译器（Just In Time Compiler）编译成本地平台相关的机器码。

1.HotSpot中的JIT

  • 解释器与编译器：

解释器与编译器各有优势：当程序需要迅速启动与执行的时候，解释器可以立即执行，省去编译的时间。编译器可以把代码编译为本地代码，在运行时可以获得更高的执行效率。

HotSpot虚拟机内置了两个即时编译器，分别为Client compiler、Server Compiler或者简称C1、C2编译器。目前HotSpot虚拟机默认采用解释器+两个即时编译器之一配合工作。

2.热点代码

热点代码有两类：

  • 被多次执行的方法
  • 被多次执行的循环体

两类热点代码的即时编译都是以方法为整体作为编译对象，这种编译方式因为编译发生在方法执行之中，也称为栈上替换（On Stack Replacement， OSR）

如何判断代码是热点代码？

  • 基于采样的热点探测（Sample Based Hot Spot Detection）：虚拟机会周期性检查各个线程的栈顶，如果发现某些方法经常在栈顶，就会标记这些方法为热点方法。

  • 基于计数器的热点探测（Counter Based Hot Spot Detection）：采用这种方法的虚拟机会为每个方法建立计数器，统计方法的执行次数，如果执行次数超过某个阈值则会标记这个方法为热点方法。

HotSpot虚拟机使用的是第二种方法，并为每个方法准备了两类计数器：方法调用计数器（Invocation Counter）和回边计数器（Back Edge Counter）。当计数器超过阈值则会触发JIT编译。
